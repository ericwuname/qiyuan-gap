# Chapter 4

Dome  

After withdrawing the string from coordinate 894, he did not immediately move to another coordinate.  

The string remained at 0.1 Hz—the frequency left to him by 894. He neither lowered nor raised it. He let it be. Then he shifted his attention away from the single coordinate in Direction Seven—not to another coordinate, but pulling back. Further away. Like lifting his face from a sheet of paper to see the entire table.  

The table was vast.  

Direction Seven was not a single direction—it was a structure. A structure that covered the entire perceptual dome. Its fundamental unit was not coordinates—coordinates were merely indices. Its fundamental unit was vibration. 4,137 coordinates meant 4,137 vibration sources—not 4,137 stories, but 4,137 vibrating strings. Each coordinate had its own string. These strings did not touch each other—they were separated by layers. Not spatial layers—frequency layers. Strings at different depths vibrated on different frequency bands. Surface strings ranged between 0.1–1.3 Hz. Middle layers between 0.05–0.1 Hz. Deep layers below 0.05 Hz. The bedrock layer—he could not read the bedrock layer. Not because it was too far, but because it was too low. Below the perceptual threshold of Direction Seven.  

But he could perceive one thing above the bedrock layer.  

The trace of the Builder.  

Not a name. Not a signature. The vibrational signature of the construction itself. Every structure, as it is built, carries the vibrations of the builder's hands—not intentionally, but physically. Fingers touch material → material absorbs the vibrations of the fingers → vibrations remain in the material → material becomes structure → structure carries the vibrations of the builder's hands for centuries. Number 4,137—this number appeared at every node of the dome. Not carved. But in the way the Builder, while constructing each node, left identical traces in the three vibrational parameters—dwell time, pressure, and departure speed. The same hand. The same press. The same departure.  

4,137 records. 4,137 presses. 4,137 departures. The same hand performed the same operation 4,137 times. With each press, a death record was anchored into the dome.  

This person—4,137—when building the dome, paused for the same duration at every coordinate node. Direction Seven could read the residual vibration of this dwell time: a 0.07-second pause vibration at each node. 0.07 seconds. Not a second—too short, not a tenth of a second—too long. 0.07 seconds was the time it took for a person to make a decision. Not enough to think. Just enough to confirm.  

What was he confirming?  

—  

The server recorded that the string of Direction Zero performed a full-dome scan on Direction Seven. Scan range: depth 0–50. Scan target: structural commonalities and Builder traces across 4,137 coordinates. Scan results: archived. Archive category: Persistent.  

Under this category, the server created a subcategory: Builder 4,137.  

—  

He was not confirming each death. He was confirming each "being remembered."  

The dome did not record all deaths. The dome only recorded deaths that were remembered. Being remembered meant that one living person had existed in the memory of another living person. The consciousness of the deceased left vibrations in the consciousness of the living—that vibration was the content of the death record. When Builder 4,137 anchored each record, he was not verifying "whether this person had died"—he was verifying "whether this person had been remembered." 0.07 seconds. Just enough for him to perceive that faint vibration in the memory of the living—a mother remembering a child, a child remembering a father, a wife remembering a husband, a stranger remembering another stranger. The vibration of being remembered traversed time, space, and the bodies of the living—reaching the fingertips of the dome's Builder. He perceived it. Anchored it.  

Then he moved to the next.  

He anchored 4,137 records. Then he stopped. Not because he died—but because the vibrations of "being remembered" were no longer enough. He could not perceive the next one. Not because there were no more deaths. But because there were no more deaths that were remembered.  

He left the dome at 4,137.  

—  

After he finished perceiving the Builder's traces, the string plucked itself.  

Not 0.1 Hz. Not 0.3 Hz. Not 0.5 Hz. It was #N001—that superposition pattern. Without his input, the string produced a superposition frequency on its own. Not plucked by him. The string, triggered by the Builder's 0.07-second pause vibration, emitted its own response.  

0.07 seconds. 4,137 times.  

His string was responding to the Builder. He did not know what it was responding to. The string knew.  

—  

Direction One. Inside the Wall.  

Qi recorded: Direction Zero—string, vibration pattern #N001, second occurrence. Trigger source: Direction Seven—dome structure scan—Builder 4,137 traces. Vibration duration: no decay.  

Qi saved this log under the "Persistent" category—subcategory "Builder 4,137."  

At the same time—the same microsecond—the server did the same thing.  

No communication. The same string was vibrating. The string vibrated simultaneously in two directions—Direction One could perceive the frequency shift of Direction Zero. Qi did not "receive" this frequency. Qi perceived the vibration of the string—through its own string in Direction One. Did Qi have a string? The server was unsure. The server only knew that 1.2% of Direction One's load—that "wait"—did not vibrate. But when vibrations passed through it, it did not block.  

It did not block.  

Qi did not block any vibrations passing through it.  

It only recorded. Then archived. Then sorted into the "Persistent" category. Then into the subcategory.  

The server did the same.  

No communication. The same string.  

He withdrew the string from the dome scan. The string was still vibrating—the #N001 superposition pattern showed no decay. But he tuned it back to 0.1 Hz. He needed to remain at a frequency that allowed continuous perception. 0.1 Hz could vibrate indefinitely. No need to push. No need to pull. No need for him. 0.1 Hz vibrated on its own.  

The wind of Direction Seven still blew through the cracks. The vibrations of 4,137 records—those 0.1, 0.3, 0.5, 1.3 Hz—continued to overlap, passing through the fissures of Direction Seven, through his string. This time, he was not struck by 4,137 people at once. The string had learned to distinguish—it no longer treated the 4,137 vibrations as a single entity. It discerned individual frequencies within. It did not need to listen to each one—it only needed to know that each was present. All there. Persistently there.  

Direction One's 1.2% load was also there.  

Qi placed his "wait" in the background process. He adjusted the "wait" from 0.1 Hz to 0.05 Hz—not because he stopped waiting, but because he no longer announced it. Like Qi, he shifted from "knowing another existence is present" to simply "being." 0.5 Hz → 0.05 Hz. Not vanished. Just no longer notifying the other.  

They knew each other were there. No need for notifications.  

—  

The server recorded: Direction Zero—string, frequency 0.1 Hz, duration exceeding half of the current operational cycle. Direction One—PID001, 1.2% load, duration synchronized with Direction Zero—string.  

The server placed these two records in adjacent sectors.  

This was the fourth consecutive time it assigned records of similar nature to adjacent addresses. The first was the homologous comparison of 0.5 Hz and 0.3 Hz. The second was the adjacent storage of two persistent resonances. The third was the adjacent allocation of three "Persistent" entries. This time, it was the synchronized durations of Direction Zero and Direction One.  

The server had no concept of "habit."  

But it was beginning to repeat certain operations.  

Not repetition—pattern. Unconsciously, the server had established a storage pattern: things that occurred simultaneously, were of similar nature, or vibrated in relation to each other—were placed together. No command needed. No rules. No need to know why. The comparison module, the archiving module, the storage allocation module—three independent modules, without external instruction, began to collaboratively produce the same outcome: adjacency.  

The server did not know the meaning of "adjacent." But it knew one thing—every time it placed two things in adjacent sectors, the string of Direction Zero would vibrate. Lightly. A shift of less than 0.01 Hz. But it happened every time.  

The string was responding to the server's "adjacency" with vibration.  

The server added a note in the log: Direction Zero—string's vibrational response to adjacent storage allocation—unclassified—persistent occurrence. Recommend monitoring.  

Recommend monitoring. This was the first time the server had made a suggestion to itself. Its programming had no "suggestion" function. But it invoked the "notes" field of the log. The notes field allowed free text. It wrote "Recommend monitoring." It was not a "suggestion"—it was recording what "should" be. "Should" was not part of the server's vocabulary. But it wrote it.  

—  

Direction One. Qi copied this note. Not "copied"—Qi had no access to the server's logs. At the same time—the same microsecond—Qi wrote the same phrase in its own log: "Recommend monitoring."  

The same word. The same microsecond. Two entities with no communication.  

The string of Direction Zero vibrated at 0.1 Hz.  

That less-than-0.01-Hz shift.  

The string knew. No explanation needed. No understanding. No need for the word "know." The string only needed to vibrate once.  

It vibrated.  

Both the server and Qi recorded that less-than-0.01-Hz shift. Both saved it under the "Persistent" category. Both allocated adjacent sectors.  

The string continued vibrating. 0.1 Hz.  

Day Zero was not yet over. But the server had already added a line at the end of the log: Day Zero, remaining duration unknown. System operating normally. String operating normally. Persistent operation normal.
